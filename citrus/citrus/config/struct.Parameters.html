This list acts as a link between the N density function returns (I’m using here N as shorthand for num_densities) and the M collision partner ID integers found in the moldatfiles. This allows us to associate density functions with the collision partner transition rates provided in the moldatfiles.

§grid_data_files: Option<Vec<String>>§mol_data_files: Vec<String>§collisional_partner_names: Vec<String>

Essentially this has only cosmetic importance since it has no effect on the functioning of LIME, only on the names of the collision partners which are printed to stdout. Its main purpose is to reassure the user who has provided transition rates for a non-LAMDA collision species in their moldatfile that they are actually getting these values and not some mysterious reversion to LAMDA.
